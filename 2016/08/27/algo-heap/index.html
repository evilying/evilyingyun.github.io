Heap

Heap: complete binary tree

  • complete binary tree
    • every level is completely filled, except possibly the last level, and all nodes are as far left as possible.
  • full binary tree
    • each node is eigher a leaf or has degree exactly 2.

implementation

  • array
    • stored in an array A[1,2, …, n]
    • $parent(j) = j/2$
    • $leftChild(j) = 2j$
    • $rightChild(j) = 2j+1$
      • parent(), leftChild() and rightChild() are all index functions
      • why we use complete full tree as the implemented data structure, why not use general binary tree?

        only this structure can use array to build, array must be linearly stored in memory.

  • pointer (static pointer)
    • TBC

Max-heaps

defination

For each node $j$, except the root, such that $A[parent(j)] \ge A[j]$

properties

  • maximum element is root
  • minimum element must be leaves
    • not only on the last level
  • the number of nodes of height(h) Max-heap
    • $2^h < n < 2^{h+1}-1$
  • the leaves in the array is: $[n/2+1,…,n]$
  • height: height of node $i$ is the maximum number of edges on a path from $i$ to a leaf
    • height of a tree is the height of the root

bulding and using heaps

maintain max-heap property

  • largest is just the index of largest value not the value.
  • running time of Max-Heapify?
    • $T(n) \le T(2n/3) + \Theta(1)$
    • $T(n) = O(logn)$
      • the largest size of subtree of complete binary tree
      • half-full on the last level
        $$size(leftChild(i)) = 2^{h-1} + \frac{n-2^{h-1}-1}{2}
        = 2 * 2^{h-1} - 1$$
        $$n = 2^{h-1} + 2^h - 1
        = 3 * 2^{h-1} - 1$$
        $$\frac{size(leftChild(i))}{n} =
        \frac{2*2^{h-1} - 1}{3*2^{h-1} - 1} \le \frac{2}{3}$$

build a heap

  • each call to Max-Heapify(i) takes time O(height(i))
  • a heap has height $\lfloor logn \rfloor$
  • There are at most $\lceil n/2^{h+1} \rceil$ nodes at height h
  • Total running time:
    $$\sum_{h=0}^{\lfloor logn \rfloor}{\lceil \frac{n}{2^{h+1}} \rceil}O(h)
    = O(n)$$

sort using a heap

  • building a max-heap takes time $O(n)$
  • There are $n-1$ calls to Max-Heapify
  • Each call to Max-Heapify takes time $O(logn)$
  • Total running time: $O(nlogn)$
