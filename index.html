Robot evil3 environment

Connecting to the Internet via USB

  • Bug Warning in windows 7
    • When we use windows7, bug is that the evil3 can connect to the host, but cannot access the internet.
  • ssh to virtualbox, then ssh to evil3
    • The best way to login to a guest Linux VirtualBox VM is port forwarding.
    • By default, you should have one interface already which is using NAT.
    • Then go to the Network settings and click the Port Forwarding button.
    • Add a new Rule:
      Host port 3022, guest port 22, name ssh, other left blank.
    • Please be sure you don’t forget to install an SSH server:
      sudo apt-get install openssh-server
    • To SSH into the guest VM, write:
      ssh -p 3022 user@127.0.0.1
      Where user is your username within the VM.

Set up cross-compiling environment

  • docker on ubuntu 16.04.1 LTS as my cross-compiling environment
    • install docker
    • Warning: need to install qemu-user-static on your host computer, otherwise you will get an error: exec user process caused “exec format error”.

Errors I meet

  • /usr/include/features.h:367:25: fatal error: sys/cdefs.h: No such file or directory
    • Sol: need to apt-get install libc6-dev-i386
  • ld: cannot find crti.o: No such file or directory
    • Bug in Ubuntu16
      • In fact not this problem
    • Sol: Just one error in the ev3dev tutorial
      • no –sysroot settings
