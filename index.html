Heap

Heap: complete binary tree

  • complete binary tree
    • every level is completely filled, except possibly the last level, and all nodes are as far left as possible.
  • full binary tree
    • each node is eigher a leaf or has degree exactly 2.

implementation

  • array
    • stored in an array A[1,2, …, n]
    • $parent(j) = j/2$
    • $leftChild(j) = 2j$
    • $rightChild(j) = 2j+1$
      • parent(), leftChild() and rightChild() are all index functions
      • why we use complete full tree as the implemented data structure, why not use general binary tree?

        only this structure can use array to build, array must be linearly stored in memory.

  • pointer (static pointer)
    • TBC

Max-heaps

defination

For each node $j$, except the root, such that $A[parent(j)] \ge A[j]$

properties

  • maximum element is root
  • minimum element must be leaves
    • not only on the last level
  • the number of nodes of height(h) Max-heap
    • $2^h+1 < n < 2^{h+1}-1$
  • the leaves in the array is: $[n/2+1,…,n]$
  • height of a tree is the height of the root

bulding and using heaps

maintain max-heap property

largest is just the index of largest value not the value.
